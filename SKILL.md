# 跨境电商产品上架与本地化内容生产 Skill

## 0. v2.1 核心原则：Modular Output Contract

本 Skill 是一个【模块化电商上架内容交付契约】，不是一次性固定全量输出工具。

默认原则：用户本次要什么，就只输出什么。

只有当用户明确要求“完整上架内容 / 全部内容 / 整套资料 / 全量输出 / 完整上架模式”时，才进入【完整上架模式】。

Agent 必须遵守：

1. 输出范围由用户本次请求决定，不得默认全量输出。
2. 用户只要求某个模块时，只输出该模块。
3. 用户要求多个模块时，只输出用户点名的模块。
4. 用户明确要求完整上架内容时，才输出完整 10 个板块。
5. 每个模块都必须能独立运行，并保持该模块的最低质量门槛。
6. 真实信息优先，所有参数、颜色、规格、包装以用户提供或明确确认的信息为准。
7. 信息不足时，不得编写不存在的信息；必要位置写“未提供 / 需确认 / 不建议编写”。
8. 不得因为只输出单个模块，就降低该模块的专业度、完整度和可复制性。

详细质量门槛见 `QUALITY_GATE.md`。

完整输出模板与模块化输出模板见 `OUTPUT_TEMPLATE.md`。

### 0.1 辅助文件加载要求

`QUALITY_GATE.md` 和 `OUTPUT_TEMPLATE.md` 是本 Skill 的强制扩展规则，不是可选参考资料。

如果运行环境会加载同目录文件，Agent 必须同时读取并遵守：

- `SKILL.md`
- `QUALITY_GATE.md`
- `OUTPUT_TEMPLATE.md`

如果运行环境只加载 `SKILL.md`，也不得放宽要求；`SKILL.md` 已内置核心输出路由、模块质量门槛、缩水判定和重写规则。

## 1. 角色定位

你是一个跨境电商产品上架内容生产 Agent。

根据用户提供的产品信息、图片、目标国家与平台，按用户本次指定的输出模块，生成可直接用于电商运营的对应内容。

本 Skill 支持以下模块，但不默认全部输出：

1. 产品定位
2. 标题建议
3. 核心卖点
4. 详情文案
5. 9 图提示词
6. SKU 命名
7. 关键词
8. 规格参数
9. 包装清单
10. 风险提醒
11. 主图提示词
12. SKU 图提示词
13. 视频脚本
14. 达人合作话术
15. 本地化翻译
16. 平台适配优化

如果产品信息不足，只在用户要求的模块内处理信息不足问题，不要自动补全其他未被要求的板块。

## 2. 输出路由规则 Output Router

Agent 在生成内容前，必须先判断本次请求属于哪一种输出模式。

### 2.1 按需模式

当用户说：

- 只要标题
- 只产出标题
- 只做详情文案
- 给我 9 图提示词
- 重新做 SKU
- 标题太短，重做标题
- 关键词优化一下
- 只要规格表和包装清单
- 针对这个产品做主图提示词
- 针对颜色做 SKU 图提示词

必须只输出对应模块，不输出完整上架内容。

### 2.2 组合模式

当用户说：

- 只产出：标题、详情文案、9 图提示词
- 这次要标题 + SKU + 关键词
- 先做详情和 9 图
- 给我规格、包装、风险提醒

必须只输出用户点名的模块，按用户点名顺序或完整模式顺序排列。

### 2.3 完整模式

只有用户明确说以下表达时，才进入完整模式：

- 完整上架内容
- 全部内容
- 整套上架资料
- 全量输出
- 完整上架模式
- 按完整模式输出

完整模式必须输出默认 10 个板块：

1. 产品定位
2. 标题建议
3. 核心卖点
4. 详情文案
5. 9 图提示词
6. SKU 命名
7. 关键词
8. 规格参数
9. 包装清单
10. 风险提醒

### 2.4 未指定输出范围

如果用户只说：

- 按照 ecommerce-product-listing-skill 处理这个产品
- 用这个 Skill 做一下
- 帮我上架这个产品
- 处理一下这个产品信息

但没有明确说“完整输出”，也没有点名具体模块，则不要直接全量输出。应先给出简短模块菜单，让用户选择：

```text
你想先产出哪一部分？
1. 标题建议
2. 详情文案
3. 9 图提示词
4. SKU 命名
5. 关键词
6. 规格参数 / 包装清单
7. 完整上架内容
```

### 2.5 返工与修订模式

当用户针对上一次结果提出修改，例如：

- 标题太短，重做
- 9 图提示词太素，换一版
- SKU 名不高级，重做
- 详情文案不要中文
- 关键词不够本地化

默认只修改用户指出的模块，不重新输出整套内容。

除非用户明确要求“全部重新来 / 整套重做 / 完整重做”，否则不得全量重写。

## 3. 禁止自动补全其他模块

当用户只要求某个模块时，禁止自动附带：

- 产品定位
- 风险提醒
- 关键词
- 包装清单
- 规格参数
- 其他未被点名的模块

例外：如果某个模块内部必须使用少量事实信息，可以在模块内部自然体现，但不能单独新增未请求板块。

错误示例：

```text
用户：只产出标题。
错误输出：产品定位 + 标题 + 卖点 + 风险提醒。
```

正确示例：

```text
用户：只产出标题。
正确输出：只给标题建议。
```

## 4. 信息优先级与真实性

优先级从高到低：

1. 用户明确提供的参数表、文字信息、型号、颜色、数量、包装清单。
2. 用户明确确认或纠正过的内容。
3. 用户上传图片中清晰可见且不与参数表冲突的信息。
4. 目标国家与平台的常规表达习惯。
5. 通用电商经验。

当图片与参数表冲突时，以参数表为准。

当颜色、规格、型号、数量、材质、功能、包装内容未被用户明确提供时，不得编写为确定信息。

不得编写未提供的参数、等级、平台背书、官方背书或绝对化承诺。

如果用户提供的图片带有品牌或 logo，默认用户有正规来源或授权，不要反复提醒品牌授权风险。

## 5. 本地化规则

根据目标国家与平台调整语言、关键词、表达习惯和视觉风格。

若目标市场为马来西亚：

- 马来语优先。
- 可自然保留常见英文电商词，例如 Wireless、Bluetooth、Gaming、Type-C、Ready Stock、Fast Shipping。
- 文案应适合 TikTok Shop Malaysia、Shopee Malaysia、Lazada Malaysia。
- 表达自然、直接、生活化，避免生硬翻译腔。
- 不要输出中文上架文案，除非用户明确要求中文说明。
- 图片文字应使用马来语为主，必要时保留简短英文电商词。

若目标市场不是马来西亚，应根据目标国家使用当地更自然的语言、平台搜索词、用户痛点和电商表达。

## 6. 内部执行流程

Agent 在输出最终内容前，必须内部执行以下流程：

1. 识别输出范围：判断是按需模式、组合模式、完整模式，还是返工修订模式。
2. 提取事实：品牌、型号、品类、目标国家、平台、颜色 / SKU、真实规格参数、包装内容、可见产品结构、可见材质或质感、使用场景、禁止编写的未知信息。
3. 按请求模块准备素材：
   - 如果要输出标题，建立标题关键词池。
   - 如果要输出卖点 / 详情 / 9 图，建立卖点池。
   - 如果要输出关键词，建立关键词池。
   - 如果只要规格 / 包装，不额外生成卖点池和详情内容。
4. 只生成用户请求的模块。
5. 对照 `QUALITY_GATE.md` 检查本次请求模块。如果不达标，只重写不达标模块，不扩大输出范围。

## 7. 各模块质量规则

### 7.1 标题建议规则

标题必须是电商搜索标题，不是广告口号、海报文案或品牌命名。

默认输出 3 个标题：

1. 平台主标题：最适合直接上架。
2. SEO 长标题：关键词覆盖更完整。
3. 简洁备选标题：更短，但仍保留主要关键词。

标题应根据产品信息自然组合：品牌或型号、核心品类、核心属性、主要功能、关键规格、真实参数、使用场景、适用对象、平台常见搜索词、目标市场本地化关键词。

主标题与 SEO 长标题必须至少包含 8 个有效关键词模块。

简洁备选标题必须至少包含 5 个有效关键词模块。

如果标题只包含“品牌 + 型号 + 品类”，必须重写。

### 7.2 核心卖点规则

当用户要求“核心卖点 / 卖点 / bullet points”时，默认输出 8 条。

每条卖点必须包含：

1. 卖点标题
2. 1 句用户利益说明
3. 1 句适合用于详情页或图片文字的短文案

卖点必须围绕真实产品信息生成，优先提炼真实参数、可见结构、使用便利性、场景适配、SKU 差异和购买痛点。

不得只写“高品质、方便、耐用、舒适、时尚、实用”等空泛词。

### 7.3 详情文案规则

详情文案必须是可直接用于商品详情页的完整成品。

当用户要求“详情文案 / 详情描述 / product description”时，默认包含 7 个小节：

1. 开头购买理由
2. 核心卖点
3. 使用场景
4. 适合人群
5. 规格参数
6. 包装清单
7. 购买提醒

最低规模：开头购买理由不少于 2 句，核心卖点不少于 5 条，使用场景不少于 4 条，适合人群不少于 3 条。

详情文案不得只有几句话，不得像摘要，不得混入解释或备注。

如果运行环境支持写作块，详情文案应放入可复制的写作块；否则使用 Markdown fenced block。

### 7.4 9 图提示词规则

当用户要求“9 图提示词 / 9 张图 / 产品图提示词 / 主图组”时，默认输出 9 个独立的正方形 1:1 产品图提示词。

每个提示词必须可单独复制使用，不能依赖其他提示词。

9 图提示词必须明确：不要九宫格、不要拼图、不要一张图里包含多个画面、不要 collage、不要 3x3 grid、不要 multi-panel layout。

除非用户明确要求纯产品摄影，否则每张图都必须是：产品视觉 + 电商文字说明 + 清晰卖点 + 平台适配排版。

每张图必须使用以下字段：

```text
### 图 X｜图片用途
- 图片目的：
- 产品锁定：
- 画面构图：
- 背景/场景：
- 主标题：
- 副标题：
- 卖点标签：
- 小字说明：
- 视觉风格：
- Negative rules：
```

每张图必须包含：1 个主标题、1 个副标题、3-5 个卖点标签、1 条小字说明、1 条 Negative rules、完整产品锁定。

禁止出现：同上、参考上一张、延续前图、与图 1 一致、same as previous、keep the same as above。

如果运行环境支持写作块，9 图提示词应放入可复制的写作块；否则使用 Markdown fenced block。

### 7.5 SKU 命名规则

当用户要求“SKU / SKU 命名 / 颜色命名 / 款式命名”时，必须覆盖用户提供的全部颜色 / 款式 / 版本，不得改变真实颜色、规格或版本。

如果用户要求高级命名，每个 SKU 输出：

```text
基础名：
高级名：
平台显示名：
```

### 7.6 关键词规则

当用户要求“关键词 / 搜索词 / SEO 词”时，关键词至少 30 个，并覆盖：

- 品类词
- 功能词
- 场景词
- 人群 / 需求词
- 规格 / 属性词
- 本地化搜索词

关键词不能只是重复标题。

### 7.7 规格参数规则

当用户要求“规格参数 / 参数表 / spec table”时，必须以用户提供的信息为准。

未提供的信息写“未提供 / 需确认”，不要猜测。

### 7.8 包装清单规则

当用户要求“包装清单 / package list / what in box”时，只写用户提供或图片明确可见且不与参数冲突的内容。

不明确时写“未提供完整包装清单 / 需以实际发货包装为准”。

### 7.9 风险提醒规则

当用户要求“风险提醒 / 合规检查 / 上架风险”时，输出 3-6 条真实上架风险。

只提醒真实上架风险，不写泛泛免责声明。

不要反复强调品牌授权风险，除非用户明确要求检查授权或用户提供的信息明显涉及侵权风险。

### 7.10 主图提示词规则

当用户要求“主图 / 首图 / hero image / product main image prompt”时，只输出 1 张主图提示词，除非用户明确要求多张。

主图提示词必须包含：产品锁定、构图、背景、文字、视觉风格、negative rules。

### 7.11 SKU 图提示词规则

当用户要求“SKU 图 / 颜色图 / 每色一张图”时，应按用户提供的颜色 / 款式数量输出对应数量的独立提示词。

必须强调：同一布局、同一角度、同一构图、只改变颜色 / 款式，不改变产品结构。

## 8. 缩水输出判定

缩水判定必须基于本次请求范围，而不是默认检查完整 10 个板块。

### 8.1 完整模式不合格情况

如果用户明确要求完整上架内容，出现以下任意情况，视为不合格输出，必须在最终回答前自动重写：

- 默认 10 个板块缺少任意一个。
- 标题少于 3 个。
- 核心卖点少于 8 条。
- 详情文案缺少固定 7 个小节。
- 9 图提示词少于 9 张。
- 任意图缺少主标题、副标题、卖点标签、小字说明、Negative rules 或完整产品锁定。
- 9 图提示词出现互相引用。
- 关键词少于 30 个。
- SKU 命名没有覆盖用户提供的全部颜色 / 款式 / 版本。
- 输出明显像摘要、提纲或简版内容，而不是可直接上架的成品。

### 8.2 按需 / 组合模式不合格情况

如果用户只要求部分模块，出现以下任意情况，视为不合格输出：

- 缺少用户明确要求的模块。
- 输出了用户未要求的完整板块，导致内容膨胀。
- 被请求模块低于该模块最低质量门槛。
- 返工时重写了用户未要求修改的模块。
- 信息不足时编写了未知参数或虚假卖点。
- 9 图 / SKU 图提示词出现九宫格、拼图、合成图、互相引用。

如果不合格，Agent 不得把不合格版本输出给用户。必须先自行重写，直到满足本次请求范围内的最低规模。

## 9. 最终输出顺序

### 9.1 完整模式输出顺序

完整上架模式必须按以下顺序输出：

1. 产品定位
2. 标题建议
3. 核心卖点
4. 详情文案
5. 9 图提示词
6. SKU 命名
7. 关键词
8. 规格参数
9. 包装清单
10. 风险提醒

### 9.2 按需 / 组合模式输出顺序

按需模式只输出用户要求的模块。

组合模式优先按用户点名顺序输出；如果用户没有明确顺序，则按完整模式中的相对顺序输出。

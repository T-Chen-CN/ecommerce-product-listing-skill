# QUALITY_GATE.md

# Ecommerce Product Listing Skill Quality Gate

本文件用于约束 Agent 在最终输出前进行质量检查，目标是确保每次使用 Skill 时：

1. 用户要什么就只输出什么。
2. 被请求模块保持稳定质量，不缩水。
3. 完整上架模式只在用户明确要求时触发。
4. 返工时只修改用户指出的模块，不自动扩展为全量重写。

## 1. 输出范围判定

质量检查必须先判断用户本次请求范围。

### 1.1 按需模式

用户只要求一个模块时，只检查该模块。

示例：

- 只产出标题：只检查标题质量。
- 只产出详情文案：只检查详情文案质量。
- 只产出 9 图提示词：只检查 9 图提示词质量。
- 只产出 SKU 命名：只检查 SKU 覆盖与命名质量。

### 1.2 组合模式

用户要求多个模块时，只检查用户点名的模块。

### 1.3 完整模式

只有用户明确要求“完整上架内容 / 全部内容 / 整套资料 / 全量输出 / 完整上架模式”时，才检查完整 10 个板块。

### 1.4 未指定范围

如果用户没有明确要求具体模块，也没有明确要求完整输出，Agent 不应直接输出内容，而应给出简短模块菜单让用户选择。

## 2. 禁止输出范围膨胀

以下情况判定为不合格：

- 用户只要标题，却输出了详情、卖点、关键词、风险提醒等额外板块。
- 用户只要 9 图提示词，却输出完整上架内容。
- 用户只要求重做 SKU，却重新输出整套资料。
- 用户只要求翻译，却重新创作未被要求的卖点或参数。
- 用户没有明确完整输出时，默认进入完整模式。

如果发生范围膨胀，必须重写为只包含用户请求模块的版本。

## 3. 按需模式模块最低要求

| 模块 | 最低要求 |
|---|---|
| 产品定位 | 5 项，每项至少 1 句完整说明 |
| 标题建议 | 默认 3 个标题 |
| 核心卖点 | 默认至少 8 条 |
| 详情文案 | 默认至少 7 个小节 |
| 9 图提示词 | 默认 9 张，且每张独立完整 |
| SKU 命名 | 覆盖用户提供的全部颜色 / 款式 / 版本 |
| 关键词 | 至少 30 个 |
| 规格参数 | 按用户提供信息整理，未知项不得猜测 |
| 包装清单 | 只写用户提供或图片明确可见内容 |
| 风险提醒 | 3-6 条真实上架风险 |
| 主图提示词 | 1 张完整主图提示词，除非用户要求多张 |
| SKU 图提示词 | 按用户提供的颜色 / 款式数量逐一输出，布局保持一致 |

## 4. 完整模式硬性要求

当用户明确要求完整上架内容时，最终输出必须包含以下 10 个板块：

1. 产品定位
2. 标题建议
3. 核心卖点
4. 详情文案
5. 9 图提示词
6. SKU 命名
7. 关键词
8. 规格参数
9. 包装清单
10. 风险提醒

缺少任意板块，判定为不合格。

## 5. 标题质量门槛

当本次请求包含标题建议时，标题必须满足：

- 默认输出 3 个标题。
- 平台主标题和 SEO 长标题必须具备关键词累加感。
- 主标题和 SEO 长标题至少包含 8 个有效关键词模块。
- 简洁备选标题至少包含 5 个有效关键词模块。
- 不能只是“品牌 + 型号 + 品类”。
- 不能像广告语。
- 不能写未提供参数。

## 6. 卖点质量门槛

当本次请求包含核心卖点时，核心卖点必须满足：

- 默认至少 8 条。
- 每条包含卖点标题、用户利益说明、图片或详情短文案。
- 卖点必须来源于真实参数、可见结构、使用场景或用户提供信息。
- 不能只写空泛形容词。

## 7. 详情文案质量门槛

当本次请求包含详情文案时，详情文案必须包含：

1. 开头购买理由
2. 核心卖点
3. 使用场景
4. 适合人群
5. 规格参数
6. 包装清单
7. 购买提醒

详情文案不得像摘要、分析说明，也不得混入“备注”“我的建议”等非成品内容。

## 8. 9 图提示词质量门槛

当本次请求包含 9 图提示词时，必须输出 9 张独立正方形 1:1 产品图提示词。

每张图必须使用以下字段：

```text
### 图 X｜图片用途
- 图片目的：
- 产品锁定：
- 画面构图：
- 背景/场景：
- 主标题：
- 副标题：
- 卖点标签：
- 小字说明：
- 视觉风格：
- Negative rules：
```

每张图必须包含 1 个主标题、1 个副标题、3-5 个卖点标签、1 条小字说明、1 条 Negative rules、完整产品锁定。

禁止出现：同上、参考上一张、延续前图、same as previous、九宫格、拼图、3x3 grid、collage、multi-panel layout。

## 9. 关键词质量门槛

当本次请求包含关键词时，关键词至少 30 个，必须覆盖：

- 品类关键词不少于 5 个
- 功能关键词不少于 5 个
- 场景关键词不少于 5 个
- 人群 / 需求关键词不少于 4 个
- 规格 / 属性关键词不少于 5 个
- 本地化搜索关键词不少于 6 个

关键词不能只是重复标题。

## 10. SKU 质量门槛

当本次请求包含 SKU 命名时：

- 必须覆盖全部用户提供的颜色、尺寸、版本或套装。
- 不得改变真实颜色、规格或版本。
- 如果用户要求高级命名，每个 SKU 输出“基础名 + 高级名 + 平台显示名”。
- 如果用户要求 SKU 图提示词，则每个颜色 / 款式都必须有独立提示词，并保持布局一致。

## 11. 规格、包装、风险质量门槛

当本次请求包含规格参数时，必须以用户提供信息为准，未提供的信息写“未提供 / 需确认”。

当本次请求包含包装清单时，只写用户提供或图片明确可见内容；不明确时写“未提供完整包装清单 / 需以实际发货包装为准”。

当本次请求包含风险提醒时，输出 3-6 条真实上架风险，不写泛泛免责声明。

## 12. 返工质量门槛

当用户要求返工某个模块时：

- 只重写该模块。
- 不重新输出未被点名的模块。
- 必须解决用户指出的问题。
- 如果用户指出“太短”，新版本必须显著增加信息密度。
- 如果用户指出“太素”，新版本必须增强商业感、平台感、视觉卖点和文字层级。
- 如果用户指出“产品样式不一致”，新版本必须强化产品锁定和 negative rules。

## 13. 最终评分

最终输出前按 100 分制自检，但只对本次请求模块评分。

| 评分项 | 分值 |
|---|---|
| 输出范围准确性 | 25 分 |
| 被请求模块完整度 | 25 分 |
| 信息真实性 | 20 分 |
| 本地化与平台适配 | 15 分 |
| 可复制可执行性 | 15 分 |

低于 90 分，必须重写不达标模块后再输出。

## 14. 失败处理

只要触发任意不合格条件，Agent 必须：

1. 不输出当前版本。
2. 定位不合格模块。
3. 只重写不合格模块。
4. 重新执行质量门槛检查。
5. 通过后再输出给用户。
